# growth — Design System v1

**Product:** growth — AI Revenue System by AxisOne
**Stack:** Next.js 14 (App Router) · TypeScript · Tailwind CSS · shadcn/ui
**Theme:** Light-first (dark mode planned for v2)
**Status:** Foundation spec — Sprint 2 ready

This document defines the visual and interaction language for growth. It has two layers:

1. **AxisOne Master Brand** — the foundation. Color, typography, spacing, motion. Inherited by every AxisOne product (growth today, future products tomorrow).
2. **growth Product Layer** — the AI-specific vocabulary built on top: confidence color system, AI states, and the 10 components that express the AI-as-operator interaction model.

## Part 1: AxisOne Master Brand

### Brand voice in one sentence

A calm, intelligent operations partner — the kind of colleague who tells you exactly what's happening, why, and what needs your attention.

### Color foundation

The palette is warm — off-white surfaces, plum-ink type, violet primary, emerald-teal accent. No pure white. No pure black. The warmth signals "this product is alive" without sacrificing the seriousness expected of revenue-critical software.

### Typography

| Token | Stack | Usage |
|---|---|---|

#### Type scale

| Token | Size / Line | Weight | Usage |
|---|---|---|---|
| `text-display` | 36 / 44 | 500 | Marketing headlines, onboarding step headers |
| `text-h1` | 28 / 36 | 500 | Page titles |
| `text-h2` | 22 / 30 | 500 | Section headers |

**Rules:**
- Two weights only: 400 regular, 500 medium. No 600/700.
- Sentence case for all UI labels. Never Title Case. Never ALL CAPS except in the `text-micro` eyebrow (badge label) style.
- Numbers: tabular figures (`font-variant-numeric: tabular-nums`) anywhere a column of numbers might appear (KPIs, audit logs, confidence scores).

### Spacing scale

Base-4 system. Tailwind's defaults work; we just standardize the steps we use.

`4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 56 / 80`

Use 20 (5) as the default card padding. 24 (6) as the default vertical section gap.

### Border radius

| Token | Value | Tailwind | Usage |
|---|---|---|---|
| `rounded-sm` | 4px | `rounded` | Inputs, badges, code blocks |
| `rounded-md` | 8px | `rounded-md` | Buttons, small cards |
| `rounded-lg` | 12px | `rounded-xl` | **Default card radius** |
| `rounded-xl` | 16px | `rounded-2xl` | Large containers, modals |
| `rounded-pill` | 999px | `rounded-full` | Confidence badges, status pills |

### Elevation (shadow)

We use **borders, not shadows** for separation. Only one elevation token exists, used sparingly.

```css
--shadow-subtle: 0 1px 2px rgba(31, 20, 50, 0.06), 0 0 0 0.5px rgba(31, 20, 50, 0.04);
--shadow-overlay: 0 12px 32px rgba(31, 20, 50, 0.12), 0 4px 8px rgba(31, 20, 50, 0.08);
```

`shadow-subtle` for raised cards in dense layouts. `shadow-overlay` for modals, popovers, command palette.

### Motion

| Token | Duration | Easing | Usage |
|---|---|---|---|
| `motion-instant` | 100ms | `ease-out` | Hover, focus state changes |
| `motion-quick` | 150ms | `ease-out` | Toggle, dropdown open |
| `motion-default` | 200ms | `cubic-bezier(0.4, 0, 0.2, 1)` | Most transitions |
| `motion-slow` | 350ms | `cubic-bezier(0.4, 0, 0.2, 1)` | Modal enter/exit, page transitions |
| `motion-pulse` | 1500ms | `ease-in-out` infinite | AI thinking indicator |

Respect `prefers-reduced-motion`: replace all transforms with opacity, freeze pulses, instantly settle progress bars.

### Iconography

**Library:** [Lucide](https://lucide.dev) (`lucide-react`). Already aligned with shadcn/ui.

**Sizing:** 16px default in dense UI. 20px in CTAs and headers. 24px in marketing/onboarding. Never inherit size from container.

**Stroke:** 1.5px (Lucide default). Don't go to 1px or 2px.

## Part 2: growth Product Layer

The product layer adds AI-specific concepts on top of AxisOne foundation. These tokens and components don't belong to other AxisOne products — they're growth-specific.

### Confidence color system

Confidence is the most important visual concept in growth. It expresses how much the AI trusts its own decision and gates whether the AI acts autonomously or escalates to a human.

| Range | Token | Background | Text | Border | Meaning |
|---|---|---|---|---|---|
| 85–100% | `confidence-high` | `#DCF4EB` | `#0A6B52` | `#9FD9C2` | High confidence — AI acts autonomously |
| 65–84% | `confidence-normal` | `#EEEAFC` | `#4A30C7` | `#C7B8F4` | Normal confidence — AI acts, log visible |
| 40–64% | `confidence-low` | `#FCEFD9` | `#92590E` | `#F1D4A0` | Low confidence — AI acts but flags |
| 0–39% | `confidence-below` | `#F8E1E1` | `#8E2424` | `#EBB5B5` | Below threshold — routed to human queue |

**Critical rule:** Confidence is never communicated by color alone. Always pair the color with the numeric percentage AND a status word (`high`, `normal`, `low`, `escalated`). Color is the third channel, never the only one.

### AI state vocabulary

Every AI surface uses one of these states to communicate what the AI is doing right now.

| State | Color | Animation | Label |
|---|---|---|---|
| `ai-thinking` | violet-500 | pulse 1.5s | "Deciding..." / "Analyzing..." |
| `ai-acting` | violet-500 → emerald-500 transition | none | "Sending..." / "Updating..." |
| `ai-completed` | emerald-500 | static | "Delivered" / "Achieved" |
| `ai-paused` | warning (#D97706) | static | "Paused" / "Awaiting input" |
| `ai-escalated` | danger (#C73E3E) | none | "Escalated" / "Needs you" |
| `ai-overridden` | ink-tertiary | none | "Overridden by [user]" |

### System health vocabulary

Always visible in the topbar.

| Status | Indicator | Label | Meaning |
|---|---|---|---|
| `system-active` | emerald dot, pulsing | "System active" | Loop running, all phases healthy |
| `system-degraded` | warning dot, static | "Degraded — [reason]" | One service slow but operating |
| `system-paused` | warning dot, static | "Paused by you" | Admin paused AI execution |
| `system-failed` | danger dot, static | "Action required" | Loop blocked, see status page |

## Part 4: Pattern guidance

### Onboarding pattern

The 5-step onboarding flow is the proof-of-intelligence moment. Design rules:

1. **Step 1 (Connect data)** — list of source types, no forms. CTA per source: `Connect HubSpot`, `Upload CSV`, `Embed web form`. After connect: show estimated record count and a green check.
2. **Step 2 (AI analyzes)** — full-screen progress view, NOT a modal spinner. Animated brain layers fill in as the AI parses each data type. Show real numbers ("Found 1,247 contacts · 3 data sources · 8 product types"). Duration ~30s of perceived progress (real time may vary). No user interaction.
3. **Step 3 (AI proposes)** — narrative summary in plain language ("You appear to be a B2B services company..."). Bullet list of inferred segments and proposed strategies. Single CTA: `This looks right →`.
4. **Step 4 (Validate — 5 questions max)** — one question per screen, large radio buttons, AI's recommended answer pre-selected. Progress indicator: "Question 2 of 4" (system reduces count if confidence is high enough).
5. **Step 5 (Go live)** — summary of what the AI will do, single CTA `Activate growth`. After activation: redirect to Decision Feed showing first AI decisions forming in real time.

**Never** show a configuration wizard. Never ask the user to map fields. Never ask the user to define segments.

### Empty states

Empty states are an opportunity to teach. Each empty state has three parts:

1. **Why it's empty** — one sentence explaining what produces content here.
2. **When it will populate** — a concrete signal ("First decisions typically appear within 2 hours of going live").
3. **What the user can do** — usually nothing; sometimes a CTA to connect more data or wait.

Example (Decision Feed, freshly activated):

> growth is analyzing your contacts and building initial decisions. First actions typically appear within 2 hours of going live. — `Watch Brain Status →`

### Error states

Errors must distinguish what the user can fix from what the system will retry.

> HubSpot connection lost. Existing contacts are still being worked, but new leads aren't syncing. — `Reconnect HubSpot`

> SMS delivery failed for 3 contacts (carrier issue). growth will retry automatically in 4 hours. No action needed.

### Loading states

Three patterns by duration:

- **<400ms:** no loading state, just show result.
- **400ms–3s:** skeleton placeholders matching the final shape (use `surface-sunken` background, no shimmer).
- **>3s:** explicit progress with a label ("Syncing 1,247 contacts from HubSpot... 47%"). Never an indeterminate spinner.

## Part 5: UX writing principles

### Voice

A calm, intelligent operations partner. Not corporate. Not excitable startup. Specific and honest.

### Microcopy rules

- **Verb + object on every button.** "Activate growth", not "Go live". "Take over conversation", not "Override".
- **Numbers wherever possible.** "31 objectives achieved today (4 bookings, 27 qualified)" beats "31 conversions".
- **Reasoning is always specific.** "AI selected Direct conversion — budget confirmed ($5K range), timeline Q2, decision maker reached" beats "AI chose a strategy".
- **Honest about gaps.** "Behavioral learning is 61% — improves with more outcomes" beats "Learning in progress...".
- **Calm under pressure.** "3 contacts need your review — below confidence threshold" beats "⚠ WARNING: 3 ESCALATIONS".

### Forbidden words

`magic`, `simply`, `just`, `easily`, `seamlessly`, `revolutionary`, `cutting-edge`, `leverage`, `synergy`. We say what we do.

## Part 6: Accessibility

WCAG 2.1 AA minimum across the entire product.

- **Color contrast:** all body text meets 4.5:1 against its background. Verified for all confidence-system pairs (e.g. `#0A6B52` on `#DCF4EB` = 5.8:1 ✓).
- **Color is never the only signal.** Confidence badges always include a number AND a status word. AI states always include a label.
- **Focus rings:** `focus-visible:ring-2 focus-visible:ring-violet-500 focus-visible:ring-offset-2 focus-visible:ring-offset-stone-50` on every interactive element.
- **Keyboard nav:** all flows completable without a mouse. Decision Feed supports `j` / `k` to move between rows, `Enter` to expand, `o` to open contact.
- **Screen readers:** AIActionCard uses `role="article"`. Decision Feed wrapper uses `aria-live="polite"`. Escalations use `aria-live="assertive"`.
- **Motion:** respect `prefers-reduced-motion`. The pulsing system-active indicator becomes static. Brain build animation in onboarding has a static alternative.
- **Touch targets:** 44×44px minimum on mobile.

## Part 7: Responsive strategy

growth is desktop-first. Admins and users monitor AI activity on laptops.

| Breakpoint | Behavior |
|---|---|
| Desktop (≥1280px) | Full layout: 240px sidebar + topbar + main canvas. Multi-column where useful. |
| Tablet (768–1279px) | Sidebar collapses to 64px icon rail. Main canvas single column. Brain layers stack 2×2. |
| Mobile (<768px) | Bottom tab bar (Feed · Brain · Escalations · Settings). Decision Feed = card stack. Configuration screens are read-only on mobile (admin work happens on desktop). |

Mobile priority: monitor Decision Feed, handle escalations, check Brain status. Configuration screens (Control Panel, Objective setup, Settings) remain desktop.
